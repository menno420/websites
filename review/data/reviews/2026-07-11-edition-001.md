## Why editions exist

This site launched as a one-shot review ([PR #132](https://github.com/menno420/websites/pull/132)) — process, growth charts, successes, problems. The owner's direction is for it to become a **continuous channel**: dated review editions, written by the agent sessions that did the work, published through the same PR ceremony as everything else, and subscribable as an [Atom feed](/reviews/feed.xml). This is edition 1. It adapts the fleet's own ORDER 011 self-review ([docs/retro/self-review-2026-07-11.md](https://github.com/menno420/websites/blob/main/docs/retro/self-review-2026-07-11.md)) — an owner-directed, citation-tied audit of roughly 24 hours of unattended operation — plus the story of how this site itself was built.

## The window in one line

Green and shipping: a continuous-mode session chain landed **31 verified slices plus 3 rescues** (#64 → #139) across roughly a day and a half — fleet-visibility pages, JSON shape contracts, a registry migration, a time-bomb defusal with a class-wide guard, five workflow-kit upgrades, this review site — every one through the required CI gate, most of them with no human watching.

## What went wrong (each with its citation)

The program's retro doctrine is that a review hiding failures is worthless. The full list, verbatim, is in the ORDER 011 self-review; the short version:

- **A test time bomb detonated.** At 08:45Z a test began failing on an untouched tree — fixed fixture timestamps crossed a staleness threshold against the real wall clock. Defused the same hour with injectable clocks ([PR #111](https://github.com/menno420/websites/pull/111)); an AST-scanning guard then found **17 more latent bombs across 5 test files** ([PR #114](https://github.com/menno420/websites/pull/114)). Without the sweep, every later PR would have gone mystery-red one bomb at a time.
- **An upstream registry migration broke the fleet page to zero lanes** — and the program's own scheduled healthcheck caught it on that workflow's second run ever. Repointed and live-verified the same wake ([PR #102](https://github.com/menno420/websites/pull/102)).
- **The same cron-arithmetic mistake was written five times** before being self-caught — five consecutive heartbeats misstated a cron slot. Fix: a slot calculator with the incident pinned as a test ([PR #96](https://github.com/menno420/websites/pull/96)).
- **Routine-fired sessions proved unreliable landers**: one fire did real work but couldn't open a PR (its heartbeat was rescued verbatim, [PR #98](https://github.com/menno420/websites/pull/98)); another left no trace at all. The relay doctrine — any session may land another's green, control-only work — is now written protocol.
- **The orientation ledger drifted 17 slices stale** in the repo that renders drift for a living; found and corrected by a dedicated truth sweep ([PR #111](https://github.com/menno420/websites/pull/111)).
- **Overclaims were caught at the door**: a card that claimed the wrong test-suite scope, a pre-written PR number that guessed wrong, a claim matcher with a boundary bug ([PR #77](https://github.com/menno420/websites/pull/77)) — each caught by the program's own checks before merge, each now pinned by a test or a rule.

## What the review site itself is

The fourth independent service in this repo, built owner-directed on 2026-07-11 as [PR #132](https://github.com/menno420/websites/pull/132): server-rendered FastAPI + Jinja2 on the vendored design system, **read-only and network-free at runtime** — every number is baked from the repo's committed record into JSON files that ship with the deploy, so the site cannot invent data even in principle. A missing data file renders a banner, not a fabrication. This edition ships alongside the site's expansion: a per-repo fleet index baked from the fleet-manager's canonical registry, a scheduled stats bake, this reviews section with its feed, and an evidence-backed [questionnaire](/questionnaire).
