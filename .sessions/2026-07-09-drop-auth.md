# 2026-07-09 — Drop Basic auth from control-plane + dashboard (both go public)

> **Status:** `in-progress` — born-red card; will flip `complete` last.

**What this session is about (owner-directed, verbatim "Yes drop the auth"):**
Remove the HTTP Basic auth gate from the two private Railway sites
(`control-plane` = `app/`, `dashboard` = `dashboard/`) so every page serves
publicly with no credentials, and — the one security refinement — mask the
control-plane readiness board's **actions-secrets** cell to a COUNT only so the
now-public board never renders the individual GitHub Actions secret *names*
(the one genuinely non-public datum on the board). `botsite` is untouched.

## Plan

- `app/main.py` + `dashboard/app.py`: delete the `_authorized` helper + the
  `auth_gate` middleware + now-dead `base64`/`secrets` imports; keep `/healthz`.
  Apps must no longer require `SITE_PASSWORD` (no fail-closed 503).
- `app/readiness.py` + `app/templates/board.html`: secrets cell renders
  `N secret(s)` and drops the name list from the served HTML (and the
  `/api/readiness.json` payload).
- Tests: rewrite both suites for the public model; add an assertion that a
  secret NAME (e.g. `ANTHROPIC_API_KEY`) is absent from the served board HTML.
- Docs: `deployment.md`, `site.md`, `dashboard.md`, `current-state.md`, and a
  new `[D-0011]` decision entry.
- Deploy: merge → redeploy both services (auto, or pinned to merged SHA if the
  auto-deploy lags) in `superbot-websites` only; verify live.
