# 2026-07-09 — Drop Basic auth from control-plane + dashboard (both go public)

> **Status:** `complete` — auth removed from both services, board secrets cell
> masked to a count, docs + [D-0011] shipped as PR #12 (`claude/drop-auth`).
> Live prod redeploy + verification captured below.

**What this session was about (owner-directed, verbatim "Yes drop the auth"):**
Remove the HTTP Basic auth gate from the two private Railway sites
(`control-plane` = `app/`, `dashboard` = `dashboard/`) so every page serves
publicly with no credentials, plus one security refinement — mask the
control-plane readiness board's **actions-secrets** cell to a COUNT only so the
now-public board never renders the individual GitHub Actions secret *names* (the
one genuinely non-public datum on the board). `botsite` was already public and
was not touched.

## What was done

- **Auth removed (both apps).** Deleted the `_authorized` helper + the
  `auth_gate` HTTP middleware + the now-dead `base64` / `secrets` imports from
  `app/main.py` and `dashboard/app.py`. `/healthz` still works. The apps no
  longer read `SITE_PASSWORD` and no longer fail-closed 503 when it is absent —
  they serve fully public. Removed the now-dead `dashboard/config.py` (it only
  held `SITE_PASSWORD`); dropped `SITE_PASSWORD` from `app/config.py`.
- **Secrets cell masked to a count** (`app/readiness.py` + `app/templates/board.html`).
  The cell now carries `{"known", "count", "detail: 'N secret(s)'}` — never a
  `names` list. The raw GitHub secrets API response (which holds the names under
  `data.secrets[].name`) is deliberately **not** carried into the returned row, so
  the names reach neither the board HTML nor `/api/readiness.json`. Every other
  board cell is public-repo-derived and unchanged.
- **Tests** rewritten for the public model (both suites): routes now assert 200
  without credentials; dropped the 401 / bad-password / fail-closed-503 tests.
  Added `test_secret_names_never_reach_served_html` (renders the board with real
  secret names in the source data and asserts `ANTHROPIC_API_KEY` / `DATABASE_URL`
  / `ROUTINE_PAT` are absent from both the HTML and `/api/readiness.json`, and
  `3 secret(s)` is shown) + `test_secrets_cell_carries_count_not_names`.
- **Docs:** `docs/site.md`, `docs/dashboard.md`, `docs/deployment.md` now state
  both sites are public / no auth and the secrets cell is masked to a count;
  `docs/current-state.md` updated; `[D-0011]` appended to `docs/decisions.md`.
- **`SITE_PASSWORD` left set but unused in Railway** — non-destructive and
  reversible (re-adding the middleware restores the gate; the var is still set).

## 💡 Session idea

Add a tiny **public-surface leak guard** to `bootstrap.py check`: a checker that
greps the served-template + readiness output for the shapes of a real secret
(anything matching `[A-Z0-9_]{8,}` that also appears in a repo's Actions-secrets
list, or common token prefixes like `ghp_` / `xoxb-`). Going public turned "does
a secret name reach the HTML?" from a private-site non-issue into a standing
invariant — an enforcing checker (not just the one unit test added this session)
would catch a future template edit that re-introduces the names, per the repo's
"enforce, don't exhort" instinct.

## ⟲ Previous-session review

The console-feed-contract session (PR #11) did the hard thing well: it wrote the
consumer against superbot's *real* pinned contract and, in doing so, immediately
surfaced the dict-vs-list `ideas`/`bugs` defect — proof that "consume the real
shape, don't assume it" pays for itself in the same PR. What it (and every prior
websites session) could not have foreseen is what this session ran into: the
`[D-NNNN]` **stamp-discipline** checker treats *any* `D-NNNN` token — bracketed or
bare, in prose or a field — as a citation, so referencing one decision from four
docs failed `check --strict`. **Workflow improvement surfaced:** the stamp rule is
correct (one home per decision) but its failure mode is opaque — an agent naturally
cross-references a decision in every doc it touches. The kit's own
`docs/decisions.md` grammar comment should state the one-home rule inline (e.g.
"cite each `[D-NNNN]` from exactly ONE non-ledger doc; describe the state in
prose elsewhere") so the next agent designs for it up front instead of discovering
it at the checker. Captured as this session's idea-adjacent note; a router/kit
follow-up if it recurs.
