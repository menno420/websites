## ⚡ Quick reference

**Repo shape:** one repo, three independent server-rendered FastAPI services
(Python 3.12) that share code, never a process. `app/` = control-plane,
`botsite/`, `dashboard/`. Read-only toward superbot (consume its committed JSON
over raw GitHub); forward-only git.

**Run a service locally** (each is `uvicorn <pkg>.app:app`; control-plane is
`app.main:app`). None require secrets to boot — a missing `GITHUB_TOKEN` just
drops to the anonymous rate limit; a missing `SITE_PASSWORD` fails only the
control-plane `/owner*` routes closed (503), public routes still serve:

```
# from the repo root, after: pip install -r requirements.txt \
#   -r botsite/requirements.txt -r dashboard/requirements.txt python-multipart
python3 -m uvicorn app.main:app       --port 8000   # control-plane  -> /  /journal  /healthz  /owner
python3 -m uvicorn botsite.app:app    --port 8080   # botsite        -> /  /commands /features /console /healthz
python3 -m uvicorn dashboard.app:app  --port 8081   # dashboard      -> /  /functions /commands /admin(stub) /healthz
```

**Tests** (60 total; install `python-multipart` first or `app/` collection errors):

```
python3 -m pytest tests/ botsite/tests dashboard/tests -q
```

**Kit gate** (docs + session-log hygiene; the CI gate mode adds `--require-session-log`):

```
python3 bootstrap.py check --strict            # local
python3 bootstrap.py check --strict --require-session-log   # what CI runs (.github/workflows/quality.yml)
```

**Other kit commands:** `status` (mode/stance/session count), `render --live`
(fill planted-doc slots), `answer <slot> <value>` (record a slot), `ledger
--title ... --verdict ... --why ... --provenance ...` (append a `[D-NNNN]`),
`session-close` (mine reflections + rebuild `.substrate/episodic_index.json`).

**Railway:** project `superbot-websites`
(`70198ece-cbc0-484e-86d9-f8a1eca4f045`), one service per site
(`control-plane`, `botsite`, `dashboard`), each repo-connected to
`menno420/websites@main` — **merge to main IS the deploy** (no manual step;
`dashboard` sometimes lags and needs a nudge). Use `RAILWAY_API_KEY` + the
explicit `superbot-websites` IDs only.

**Docs live in `docs/`:** `current-state.md` (living ledger — read 2nd),
`decisions.md` (`[D-NNNN]` ledger), `question-router.md` (owner-intent Q-blocks),
`architecture.md` / `ownership.md` / `runtime_contracts.md` (binding contracts),
`site.md` / `botsite.md` / `dashboard.md` / `deployment.md` (per-service),
`planning/` (plans). Root: `CONSTITUTION.md` (working agreement + rails).

## Environment & boot runbook

- CI is **Python 3.12** (`.github/workflows/quality.yml`). Run tools via
  `python3 -m <tool>` to match.
- The agent container carries ambient `RAILWAY_PROJECT_ID` /
  `RAILWAY_SERVICE_ID` / `RAILWAY_ENVIRONMENT_ID` pointing at the **live
  production bot** — **never pass those to a Railway call.** Use
  `RAILWAY_API_KEY` + the explicit `superbot-websites` IDs. Never call a
  destructive Railway mutation.

## Recurring problems + fixes

- **`tests/test_app.py` collection error: "Form data requires python-multipart".**
  The `/owner` POST routes need `python-multipart` (in `requirements.txt`, but a
  bare env may lack it). `pip install python-multipart` before running tests.
- **Stamp-discipline check failure** (`D-NNNN cited from N docs`). The kit wants
  each decision id stamped at **one** home outside `docs/decisions.md`. Don't
  repeat a bare `D-NNNN` token across docs — reference the decision by effect and
  point to the ledger instead (see `docs/question-router.md`'s note).
- **`bootstrap.py check` red on a born-red session card.** An incomplete newest
  session log (missing `💡 Session idea` / `previous-session review` markers)
  fails `--strict` by design — flip the card `complete` with all three markers as
  the last step. CI picks the newest card **by mtime**, so keep *every* card
  complete to stay green regardless of checkout order.

## Past mistakes to avoid

- Merging a PR without a `.sessions/` card + a `current-state.md` Recently-shipped
  entry — the drift the kit CI gate now catches. PRs #9/#10/#13/#15 shipped
  without cards/ledger entries and had to be backfilled.
- `EXIT:$?` after a piped command (`… | tail`) reports the pipe tail's exit, not
  the real command's — check exit codes without a trailing pipe.
