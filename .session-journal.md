## ⚡ Quick reference

**Repo shape:** one repo, four independent server-rendered FastAPI services
(Python 3.12) that share code, never a process. `app/` = control-plane,
`botsite/`, `dashboard/`, `review/`. Read-only toward superbot (consume its
committed JSON over raw GitHub); forward-only git.

**Run a service locally** (each is `uvicorn <pkg>.app:app`; control-plane is
`app.main:app`). None require secrets to boot — a missing `GITHUB_TOKEN` just
drops to the anonymous rate limit; a missing `SITE_PASSWORD` fails only the
control-plane `/owner*` routes closed (503), public routes still serve:

```
# from the repo root, after: pip install -r requirements.txt \
#   -r botsite/requirements.txt -r dashboard/requirements.txt \
#   -r review/requirements.txt python-multipart
python3 -m uvicorn app.main:app       --port 8000   # control-plane  -> /  /journal  /healthz  /owner
python3 -m uvicorn botsite.app:app    --port 8080   # botsite        -> /  /commands /features /console /healthz
python3 -m uvicorn dashboard.app:app  --port 8081   # dashboard      -> /  /functions /commands /admin(stub) /healthz
python3 -m uvicorn review.app:app     --port 8082   # review         -> /  /fleet /reviews /questions /healthz
```

**Tests** (1414 total; install `python-multipart` first or `app/` collection errors):

```
python3 -m pytest tests/ botsite/tests dashboard/tests review/tests -q
```

**Kit gate** (docs + session-log hygiene; the CI gate mode adds `--require-session-log`):

```
python3 bootstrap.py check --strict            # local
python3 bootstrap.py check --strict --require-session-log   # gate mode (missing card = hard fail)
# What CI runs (.github/workflows/quality.yml) is PR-diff-AWARE — it derives the
# card the PR touches and adds --session-log <card> so a stale complete card can't
# mask THIS PR's in-progress one. To mirror CI locally on a specific card:
python3 bootstrap.py check --strict --require-session-log --session-log .sessions/<your-card>.md
```
> Engine is kit **v1.0.0** ([D-0017]): a born-red card (Status `in-progress`/`wip`/
> `hold`/`drafted`) FAILS `--strict` and is never allowlistable. Fixed the leak that
> let PR #19 auto-merge empty. Push your card + work TOGETHER (or open the card
> `complete`) so your own PR doesn't premature-merge.

## Recurring problems + fixes

- **Jinja `q.items` on a dict resolves to the `.items` METHOD, not the key.**
  Attribute lookup on a dict falls back to Python attributes, so
  `{{ q.items }}` hands the template a builtin method (TypeError at render
  when iterated/formatted). Use subscript syntax — `q['items']` — for any
  dict key that shadows a dict method (`items`, `keys`, `values`, `get`).
  Bit the ORDER 005 session live (PR #53, `queue.html`).
- **This container has `GITHUB_TOKEN` set** → `app.config.GITHUB_TOKEN` is
  truthy at import. Tests touching token-keyed behavior must monkeypatch it
  BOTH directions (set AND empty); exercising a degraded page locally needs
  `env -u GITHUB_TOKEN python3 -m uvicorn …`. Related: in-container the
  agent proxy blocks `api.github.com` for the APP too (contents-API fetches
  fail locally with a proxy error) while the raw host works — production has
  no proxy, so local "contents API down" is an environment artifact, not an
  app bug. (ORDER 005 session, PR #53.)
- **`tests/test_app.py` collection error: "Form data requires python-multipart".**
  The `/owner` POST routes need `python-multipart` (in `requirements.txt`, but a
  bare env may lack it). `pip install python-multipart` before running tests.
- **Stamp-discipline check failure** (`D-NNNN cited from N docs`). The kit wants
  each decision id stamped at **one** home outside `docs/decisions.md`. Don't
  repeat a bare `D-NNNN` token across docs — reference the decision by effect and
  point to the ledger instead (see `docs/question-router.md`'s note).
- **`bootstrap.py check` red on a born-red session card.** An incomplete session
  log (missing `💡 Session idea` / `previous-session review` markers **or** a
  Status still `in-progress`/`wip`/`hold`/`drafted`) fails `--strict` by design —
  flip the card `complete` with all three markers as the last step. As of kit
  **v1.0.0** ([D-0017]) CI is **diff-aware**: it gates on the card THIS PR touches
  (`--session-log`), not the mtime-newest one, and the Status VALUE is checked (not
  just marker presence). So an `in-progress` card can no longer sneak through, and a
  stale `complete` card can no longer mask it — but that also means your own PR's
  card must be `complete` (with work present) or the gate holds it red.

## Past mistakes to avoid

- Merging a PR without a `.sessions/` card + a `current-state.md` Recently-shipped
  entry — the drift the kit CI gate now catches. PRs #9/#10/#13/#15 shipped
  without cards/ledger entries and had to be backfilled.
- `EXIT:$?` after a piped command (`… | tail`) reports the pipe tail's exit, not
  the real command's — check exit codes without a trailing pipe.
