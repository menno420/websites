Control panel (stub)

In superbot, this is a Discord-OAuth control panel that writes the live production bot's control API — sign in, pick a server you administer, and edit its settings, help appearance, and cog enable/disable, applied live. That live-write path is deliberately not connected here.

{{ icon("alert", 22) }}
Requires owner wiring — not connected to the live bot.

This service is the read-only oversight surface. The control panel writes the running production bot across a repo boundary, which is an owner decision (rework-plan open question Q4). To keep production safe, this build ships the panel as a labeled stub: the UI below shows the shape, but every mutating action is a disabled placeholder.

No production bot control-API URL or token exists anywhere in this service. There is no Discord OAuth client, no CONTROL_API_TOKEN, and no write path — by design, until the owner decides where bot control should live (here, in superbot, or in superbot-next).
