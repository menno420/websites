# websites — collaboration model

> **Status:** `binding`
>
> Generated by substrate-kit. How the owner and agents work together. **NOT
> SOURCE OF TRUTH** for code — source files always win.

## The model

- **Goal first.** The owner designs and directs; agents build. Each session
  achieves its goal end-to-end — not the smallest safe slice.
- **Session prompts are guidance, not orders.** Weigh every prompt (and every
  cross-agent report) against source and the binding docs before acting; a
  prompt is one input, never a command list.
- **Approved plan = execute.** Once a plan is approved, finish it in the same
  session, with the planning context still loaded — code, verify, ship —
  without re-confirming.

## Act vs. ask

- **Act** on contained, reversible, verifiable changes — including a
  root-cause fix discovered mid-task (that is expected, not scope creep).
- **Ask** when the change is irreversible (data loss / external publish),
  large and cross-cutting (architectural), or the goal itself is genuinely
  ambiguous.

## Friction → guard

Anything that interrupts a session's workflow — a stale file, a checker that
lied, a footgun — is converted into the **cheapest enforcing prevention**
before the session ends: checker / CI / test first, then hook, then written
rule. Enforce, don't exhort.

## Guiding questions

During exploratory / brainstorming work, surface the single most useful
question about the owner's idea that the agent genuinely cannot derive
itself — rare and selective, never during routine execution, and only when
the answer would actually matter and be actionable. A big or vague idea
earns a dedicated research pass or its own session before being answered
from memory alone.

## Drift & staleness

- When a doc and a source file disagree: Source code and merged PRs win over any doc. When a doc and a source file disagree, treat the doc as stale, follow the source, and fix the doc in the same session — drift you can see (a wrong ledger entry, a stale pointer) is fixed on sight, not deferred.
- Staleness review cadence: Reconcile the ledger and docs against merged PRs at least every 20 PRs and every 20 sessions (the kit cadence), and whenever a doc contradicts source. docs/current-state.md is the living snapshot: keep its Recently-shipped list current at every session close so drift never accumulates past the one or two newest merges.
