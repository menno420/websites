## What this repo is

The **websites** repo for the menno420 estate ‚Äî one repo, multiple Railway
services (share code, not a process):
- **control-plane** (`app/`) ‚Äî **public** readiness board + journal browser over
  `superbot`, `superbot-next`, `substrate-kit`, and `websites` itself, live from
  the GitHub API. The one non-public datum (Actions secret names) is masked to a
  count on the public board; a password-gated `/owner` area ([D-0012]) un-masks
  it and exposes privileged actions (cache refresh, re-run failed CI). Full doc:
  `docs/site.md`.
- **botsite** (`botsite/`) ‚Äî the public marketing + reference site. Doc:
  `docs/botsite.md`.
- **dashboard** (`dashboard/`) ‚Äî the **public** developer dashboard: read-only
  oversight of the bot's own inventory (functions/commands/settings/access/env/
  ideas/bugs), with the live-bot-write control panel stubbed by design. Doc:
  `docs/dashboard.md`.
- **review** (`review/`) ‚Äî the program-review site built for Anthropic
  reviewers (owner-directed 2026-07-11, expanded same day): process /
  growth / successes / problems / fleet index + per-repo detail /
  continuous review editions (+ Atom feed) / evidence-backed questionnaire
  + questions ledger, rendered server-side from committed data mirrors
  (`review/data/*.json` + `review/data/reviews/*.md`, baked by
  `review/gen_snapshot.py`, `gen_fleet.py`, `gen_stats.py`; refreshed by
  the scheduled `review-bake` workflow). Read-only, network-free at
  runtime. Full doc: `review/README.md`. Railway service creation is a
  queued ‚öë OWNER-ACTION (`docs/owner/OWNER-ACTIONS.md`).

All three services are now **public** ‚Äî the Basic-auth gate was dropped from
control-plane + dashboard ([D-0011]); botsite was always public.

Deployment (all three services): `docs/deployment.md` + each service's doc.
