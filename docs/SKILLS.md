## What this is

The registered skill set for websites: every recurring action that
has a defined, kit-shipped procedure. **Check this index before improvising
a workflow or repo-searching "how do we do X here"** ‚Äî when a row covers
the action, invoke the skill (or read its installed body) instead of
deriving the procedure from scratch.

## Where the bodies live

- **Installed (live):** `.claude/skills/<name>/SKILL.md` ‚Äî invoke as
  `/<name>`.
- **Staged (regenerated at every adopt/upgrade):** the kit state dir's
  `skills/` tree (default `.substrate/skills/`); install with
  `python3 bootstrap.py skills --build`.
- **Precedence:** a skill's declared capability **wins over the ambient
  stance** (an invoked `session-close` may write the session log even under
  a `review` stance); stances stay advisory for anything a skill has not
  declared.

## Growing the set

The skill set is kit-owned (the `SKILLS` list in the kit's
`src/engine/skills/skills.py`) and this index regenerates from it ‚Äî never
hand-edit the table. A recurring action without a row here is a candidate
skill: capture it as an idea (`docs/ideas/README.md`) or propose it
upstream to the kit, and it reaches every adopter at the next release.
