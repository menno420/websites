## Idea lifecycle

```
captured -> classified -> planned -> built -> verified
```

Every idea ends implemented, planned, in discussion, or explicitly rejected —
never orphaned. Backlog + routing: `docs/ideas/README.md`.

## Session workflow

```
orient -> claim -> born-red card -> build -> verify -> close
```

1. **Orient** — working agreement, current state, task-specific reading route.
2. **Claim** — declare your lane so parallel sessions don't collide.
3. **Born-red card** — open the session record first, marked in-progress, so
   the work is visible while it is still incomplete.
4. **Build** — the goal, end-to-end.
5. **Verify** — run `${verify_command}` before shipping.
6. **Close** — flip the card complete; log the session, groom one idea, hand
   off.
