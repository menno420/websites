## Rules

1. **One source of truth.** A behavior lives in exactly one function. Never
   copy a helper into a second module "for convenience" — import it, or move
   it (rule 2).
2. **Shared helpers live below both consumers.** A helper needed by two
   layers goes in the shared layer *below* both — never in either consumer
   layer, and never duplicated into each.
3. **Exact-name guard.** Before defining a new function, grep for
   `def <exact_name>` in the target module and its siblings (plus the 1–2
   nearest concept synonyms). A later same-name `def` silently shadows the
   earlier one — no import fails, no warning fires.
4. **Promote on second use.** The moment a private helper is wanted by a
   second module, promote it to the shared layer — don't copy it.
